■ レシート取込・OCR・CSV化・正式経費登録構想

今後の目標として、携帯で撮影したレシート画像、またはドラッグアップロードした画像をもとに、
OCR等で読み取った内容をいったん取込データとしてDBに蓄積し、
確認・補正後に正式な経費データへ変換する仕組みを作る。

単純に画像を添付するだけではなく、以下の流れを目指す。

1. レシート画像を撮影またはアップロード
2. OCRまたは手入力CSV等により、日付・支払先・金額・税額・インボイス番号等の候補データを作成
3. 取込用の一時テーブルに保存
4. 取込一覧画面で人間が確認・修正
5. 確認済みデータを正式な経費データへ登録
6. 登録時はDBトランザクションを使い、親データ・明細データ・画像データ・取込ステータスを一括で確定
7. 失敗時はROLLBACKし、中途半端な経費データを残さない
8. 将来的に会計ソフト連携用CSV、または自社用CSVとして出力できるようにする

想定テーブル案：

expenses.import_batches
- import_batch_id
- import_type
- source_name
- original_file_name
- status
- imported_at
- memo

expenses.import_rows
- import_row_id
- import_batch_id
- source_row_no
- raw_date
- raw_vendor_name
- raw_amount
- raw_tax_amount
- raw_invoice_number
- raw_text
- suggested_expense_date
- suggested_vendor_name
- suggested_amount
- suggested_invoice_type_id
- suggested_evidence_type_id
- status
- error_message
- confirmed_expense_id
- created_at

expenses.receipt_files
- receipt_file_id
- import_row_id
- expense_id
- original_file_name
- stored_file_name
- file_path
- mime_type
- file_size
- ocr_text
- ocr_status
- created_at

重要方針：
- OCR結果を直接 expense_headers / expense_details に入れない。
- まず import_rows に仮データとして蓄積する。
- 人間が確認してから正式経費へ変換する。
- 正式登録時は transaction を使う。
- OCR、CSV取込、手入力、クレカ明細、銀行明細など、入口が違っても import_rows に集約する設計にする。

■ Wix携帯専用レシート撮影ページ・画像圧縮方針

将来的に、PCアプリ側だけではなく、Wixサイト上に携帯専用のレシート撮影・仮取込ページを作る構想がある。

目的：
- スマホでレシートを撮影する
- 撮影画像をそのまま大容量で保存・送信しない
- ブラウザ上で小さい画像へ縮小・圧縮してから保存・送信する
- 後でHD Origin Project側の取込DBへ連携する

重要方針：
- 携帯で撮影した元画像は数MB〜十数MBになる可能性があるため、そのままアップロードしない。
- スマホ容量・通信量・Wix側保存容量を圧迫しないようにする。
- レシート確認用として十分な画質を残しつつ、容量を小さくする。
- 元画像ではなく、圧縮後画像を保存対象にする。

想定仕様：
- Wix上に携帯専用ページを作る。
- 画像選択またはカメラ撮影には以下のような入力を使う想定。
  <input type="file" accept="image/*" capture="environment">
- 撮影後、JavaScriptで画像を読み込む。
- canvasを使って縮小・JPEG圧縮する。
- 圧縮後画像のみをWix側へ保存またはアップロードする。

画像圧縮の目安：
- 長辺：1200px〜1600px程度
- 形式：JPEG
- 品質：0.55〜0.70程度
- 目標容量：1枚あたりおおむね150KB〜500KB程度

想定処理：
1. スマホでWix携帯専用ページを開く
2. レシートを撮影または画像選択
3. ブラウザ内で画像を読み込む
4. canvasで長辺1200px〜1600px程度へ縮小
5. JPEG品質0.55〜0.70程度で圧縮
6. 圧縮後画像の容量を画面に表示
7. 日付・支払先・金額・メモ等を必要に応じて入力
8. 圧縮後画像と仮データをWix側へ保存
9. 後でCSVまたはAPI等によりHD Origin Project側へ取り込む
10. HD Origin Project側では import_rows 等の取込ステージに入れ、人間が確認後、正式経費へ登録する

設計上の位置付け：
- Wixは正式な経費DBではなく、スマホ撮影・仮取込の入口として使う。
- 正式な経費データは、HD Origin Project側のPostgreSQLで管理する。
- OCRやCSV取込結果は、直接 expense_headers / expense_details に入れず、まず import_rows などの取込用一時テーブルに蓄積する。
- 人間が確認・補正してから、DBトランザクションで正式経費登録する。

注意：
- Wix標準フォームのファイルアップロードだけだと、撮影画像を事前に小さくする制御が弱い可能性がある。
- 本格的にやる場合は、Wix Velo、HTML iframe、またはカスタムHTML/JSで、アップロード前にブラウザ側で圧縮する。
