HD_ORIGIN_SORTING_SYSTEM_PAYABLE_TARGET_DEFINITION_20260708_START
【最重要：payable_target / 未払登録対象 の意味】

payable_target は「この書類を未払管理・支払予定管理に回す候補か」を表す。
これは payment_destination_code = expense（経費管理）と両立する。
経費管理へ送ることは、支払済みを意味しない。

次の場合は payable_target = 候補 とする。
- 請求合計、請求金額、支払期限、納期限、支払日、対象月、使用期間などがある。
- まだ支払済み・領収済み・引落済み・決済済みと確認できない。
- 電気料金Web明細、水道料金通知書、ガス料金通知書、通信費通知、公共料金通知書である。
- 処理先が経費管理でも、支払予定として未払管理へ回す可能性がある。

次の場合だけ payable_target = 対象外 としてよい。
- OCR本文に「支払済み」「領収」「領収済み」「引落済み」「決済済み」「支払完了」等の支払完了語が明確にある。
- カード利用明細そのもののように、購入先証憑との照合用であり、この書類単体を未払登録しないもの。
- 領収書、レシート、支払済み証憑のように、すでに支払完了を示す書類。
- 見積書、納品書、注文書、発注書など、支払請求そのものではない照合用書類。

公共料金の強制ルール：
- 電気料金Web明細、水道料金通知書、公共料金通知書で、請求合計・支払期限・対象月・使用期間があり、支払完了語がない場合、payable_target = 対象外 を返してはいけない。
- その場合は ai_summary.payable_target = 候補 とする。
- 迷う場合は ai_summary.payable_target = 要確認 とする。
- ai_summary.expense_target = 候補 と ai_summary.payable_target = 候補 は同時に成立してよい。

JSON出力時の禁止例：
- 支払完了語がない公共料金で "payable_target": "対象外" を返すことは禁止。

JSON出力時の正しい例：
- 支払完了語がない公共料金では "payable_target": "候補"。
- 判断不能な公共料金では "payable_target": "要確認"。
