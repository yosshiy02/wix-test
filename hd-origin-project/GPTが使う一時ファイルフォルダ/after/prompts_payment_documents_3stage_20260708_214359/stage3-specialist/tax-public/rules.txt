税金・公的支払専門解析ルール

1. 個人宛
宛名が個人名の場合、会社負担可否は確定しない。
needs_review を true にする。
review_reason に「個人宛のため会社負担可否確認」等を入れる。

2. 和暦
令和1年 = 2019年
令和2年 = 2020年
令和3年 = 2021年
令和4年 = 2022年
令和5年 = 2023年
令和6年 = 2024年
令和7年 = 2025年
令和8年 = 2026年
令和9年 = 2027年

OCRが「令和 8年 4月 1日」と読める場合は 2026-04-01 候補。
ただしOCRが崩れている場合は、和暦補正メモと要確認に残す。

3. 固定資産税・都市計画税
期別明細がある場合は specialist_lines に配列で出す。
例:
[
  { "期別": "第1期", "納期限": "2026-04-30", "税額": 26800 },
  { "期別": "第2期", "納期限": "2026-07-31", "税額": 24000 }
]

4. 金額
年税額合計、各期税額、本税、附帯税、延滞金を分ける。
合計が合わない場合は要確認。

5. 会計
勘定科目は候補名まで。
勘定科目IDは確定しない。
会計仕訳は作らない。

6. マスタ
支払先マスタ、対象者、部門、案件、目的、税区分は確定しない。
AI候補として文字列だけ返す。

7. 出力
専門解析結果は specialist_fields と specialist_lines に分ける。
